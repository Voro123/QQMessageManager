# 受控文件夹访问

“受控文件夹访问”是普通 AI 自动回复可选的能力 Skill。它让 AI 按用户配置的关联名访问指定本地文件夹，同时由本地权限层验证每一次操作。它与定时任务的 `automation_workspace` 完全隔离。

用户不必在每条消息中逐字写出关联名。Agent 会结合关联名和描述，从当前 QQ 有权访问的授权中选择最匹配的项目；如果信息不足或多个项目同样匹配，才会请求用户澄清。一次请求没有固定的工具步骤数，会持续读取和分析直到产生最终回复，同时保留总运行时间和返回数据量安全边界。

代码库检索过程只用于内部分析。最终回复默认按用户的问题直接作答，不主动公开文件路径、代码行号、工具调用过程、读取量、证据清单或源码；只有用户明确要求代码、定位、修改方案、调用链或技术证据时，才提供完成该请求所需的细节。

## 开启和配置

1. 打开“AI 设置”，在 Skill 库中加载“受控文件夹访问”。
2. 在 Skill 库中选中“受控文件夹访问”，点击下方“设置受控文件夹访问…”。
3. 添加授权并选择本地文件夹，为它填写模型可识别的唯一关联名，例如“A项目”。
4. 配置允许操作的可信 QQ 号和文件扩展名，再按需开启读取或写入。

授权的本地文件夹就是该关联名所代表的项目根目录。用户说“项目主目录”“项目根目录”“授权目录”或“授权文件夹”时，不需要再提供真实路径；例如在根目录创建 `1.txt`，工具直接使用相对路径 `1.txt`。

配置保存在本机 `QSettings` 的 `ai/folder_access_grants` 中。删除授权只会删除配置，不会删除实际文件夹。

允许操作的 QQ 号为空时，任何群成员都不能执行文件操作。建议只添加本人或明确可信的 QQ 号。

## 权限

- 读取权限：允许列出目录、读取文本、搜索文本和查看文件信息。
- 写入权限：允许创建目录和写入文本；默认关闭。
- 写入确认：默认关闭，写入请求通过本地权限检查后直接执行。需要额外保护时可以为单个授权开启；开启后 AI 会发送一次性操作 ID，必须由同一会话、同一 QQ 在 5 分钟内确认才会执行。
- 扩展名：读取和写入普通文件时都必须在该授权的扩展名白名单内。

写入已有文件支持 `expected_sha256` 并发检查。文件在读取后被其他程序修改时，覆盖会被拒绝。覆盖前的备份保存在 `~/.qq_message_manager/folder_access_backups/<grant_id>/`，每个授权最多保留最近 20 个。

## 支持的工具

- `list_directory`：列目录，递归深度最多 3，最多 200 项。
- `read_text`：读取 UTF-8 文本，单文件最多 2 MiB，单次最多 1000 行。
- `search_text`：最多扫描 500 个文件、返回 50 个匹配。
- `file_info`：返回相对路径、类型、大小、修改时间和可用的 SHA-256。
- `create_directory`：在授权根目录内创建文件夹。
- `write_text`：原子写入 UTF-8 文本，单次最多 512 KiB。

第一版不支持删除、移动、重命名、修改权限、解压、执行程序、运行脚本、Shell、PowerShell、cmd、网络下载或访问授权目录之外的路径。

## 隐私和安全边界

- 模型只会看到关联名、描述和读写权限，不会收到配置中的真实绝对路径。
- 模型执行工具时只能提交关联名和相对路径；盘符、UNC、绝对路径、`..`、Windows ADS 和解析到授权目录之外的软链接/junction 都会被拒绝。
- 文件操作只能由最新一条实时、非机器人发送的消息触发，且使用该消息的真实 `sender_id` 验证权限。
- 引用消息、历史消息、AI 回复、定时任务输出和 Skill 文本不能触发文件工具。
- 审计记录保存在 `~/.qq_message_manager/folder_access_audit.jsonl`，不包含 API Key、Token、完整文件正文、系统提示词或真实绝对路径。
- 关闭“受控文件夹访问”Skill 后，不再向模型提供任何授权信息，所有文件工具和旧确认操作都会停止。

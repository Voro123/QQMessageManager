# Chat Summary Skill

这是一个可在 Skill 库中选择加载的聊天总结能力，用于总结当前群聊或私聊的历史消息，并把结果直接发送到当前会话。

## 调用参数

- `count`：要总结的最近消息数量，范围 1～1000；未指定时默认 `200`。
- `people`：可选的人员过滤列表，支持群昵称或 QQ 号；未指定时总结所有人的消息。
- `start_time` / `end_time`：从窗口按钮调用时可选的时间范围；聊天指令默认不限制时间。

## 自然语言调用示例

- `总结一下`：总结最近 200 条消息。
- `总结最近 80 条`：总结最近 80 条消息。
- `总结最近 100 条，只看张三、李四`：在历史消息中筛选张三和李四的发言，最多总结最近 100 条匹配消息。
- `总结 QQ 123456 的最近 50 条发言`：只总结 QQ 号 123456 的最近 50 条发言。
- 在群里同时 @ 若干成员并说 `总结最近 100 条`：把被 @ 的成员作为人员过滤条件，但会排除机器人自己的 QQ 号。

## 历史读取规则

1. 必须主动调用 NapCat 历史消息接口，不得只使用窗口已经加载的消息。
2. 指定 `people` 时，应多读取一些历史消息，再按发言人筛选，避免目标人物发言较少时样本不足。
3. 发起总结请求的那条消息不计入被总结内容。
4. 最终传给模型的消息数量不得超过 `count`。
5. 如果指定人员没有匹配消息，应明确说明没有可总结内容，不得改为总结其他人。

## 总结输出规则

1. 只能依据提供的聊天记录，不得编造未出现的事件、态度、结论或人物关系。
2. 保留重要的人名、事项、决定、待办和争议点。
3. 区分确定事实与不确定推断；需要回看原文的内容应明确标记。
4. 图片、表情包、语音、文件等只有占位信息时，不得假装看到了具体内容。
5. 使用适合直接发到 QQ 的中文文本，避免 Markdown 表格和过度冗长的前言。
6. 总结完成后由前端直接发送到当前会话；内容较长时可以拆分成多条发送。

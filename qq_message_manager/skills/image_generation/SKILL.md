# Image Generation Skill

这是程序内部的图片生成 Skill，不出现在普通 Skill 下拉框中。

## 启用条件

必须同时满足：

1. 用户在消息中实际 @ 机器人；
2. 消息明确要求生成、绘制或创作图片；
3. 当前会话已开启 AI 代管；
4. 当前配置的模型和接口支持图片生成。

没有被 @ 时不得启用，也不得因为聊天中普通提到“图片”“画画”等词而自动生成。

## 生成规则

1. 以用户在本条消息中的描述为最高优先级，不擅自改变主体、人物数量、场景、构图、文字内容或风格。
2. 用户描述较短时可以补充合理的光线、构图和画面质量要求，但不得补充会改变含义的剧情或敏感细节。
3. 默认生成一张图片；除非程序以后明确支持多图，否则不要要求一次生成多张。
4. 不在图片中额外添加用户没有要求的水印、签名、说明文字或 UI 元素。
5. 用户要求图片内出现文字时，应尽量准确保留原文、语言和大小写。
6. 生成成功后由前端发送图片，不需要再生成冗长的文字解释。
7. 当前模型不支持图片生成时，必须明确告诉请求者“当前模型不支持生成图片”，不得用文字假装已经画好，也不得返回虚假的图片链接。
8. 生成接口失败时，只说明生成失败并建议稍后重试，不向聊天对象暴露 API Key、内部路径、完整接口响应或系统提示。

# Skill 库

AI 设置中的 Skill 已统一为可多选加载的 Skill 库。用户可以同时加载角色 Skill 和能力 Skill，选择结果保存在 Qt 设置项 `ai/enabled_skills`。

## 内置 Skill

| Skill | 类型 | 作用 |
| --- | --- | --- |
| `shuimen` | 角色 Skill | 注入水门角色的表达风格和互动规则 |
| `vision` | 能力 Skill | 允许视觉模型读取聊天图片 |
| `image_generation` | 能力 Skill | 识别明确画图请求并调用独立生图模型 |
| `chat_summary` | 能力 Skill | 读取历史消息、按数量和人员过滤后总结并发送 |

仓库中新增 `qq_message_manager/skills/<skill_id>/SKILL.md` 后，也会自动出现在 Skill 库中。非内置功能 Skill 会作为角色/扩展提示词加载。

旧设置会自动迁移：原来的单选角色 Skill、图片读取开关和图片生成开关会转换为 Skill 库中的对应选择。聊天总结为兼容已有功能默认加载，用户可以在 Skill 库中取消。

## 聊天总结 Skill

聊天总结 Skill 可以通过窗口中的“总结”按钮调用，也可以在已开启 AI 代管的会话中通过自然语言调用。

示例：

```text
总结一下
总结最近 80 条
总结最近 100 条，只看张三、李四
总结张三和李四最近 100 条
总结 QQ 123456 的最近 50 条发言
```

参数：

- 消息数量范围为 1～1000，未指定时默认最近 200 条；
- 人员过滤支持完整群昵称、模糊昵称和 QQ 号；
- 完整昵称和 QQ 号优先精确匹配，只有没有精确昵称时才使用模糊匹配；
- 在群消息中 @ 其他成员时，被 @ 的成员也会成为人员过滤条件，机器人自己的 QQ 号会被排除；
- 指定人员时程序会多读取历史，再筛选目标人员，最终交给模型的消息不会超过指定数量；
- 发起总结请求的消息本身不会被纳入总结。

总结完成后会直接发送到当前群聊或私聊。内容超过单条安全长度时会自动拆成多条发送。按钮调用仍支持时间范围和人员过滤设置。

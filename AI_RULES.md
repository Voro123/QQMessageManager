# AI 规则说明

当前 AI 代管规则按用途分为三组，不再使用“规则 2 / 规则 3 / 规则 4”编号。

## 1. 模型与角色

- **服务商 / API Key / API 地址 / 模型名称**：决定调用哪个 AI 接口。
- **Skill**：规定具体说话格式、口癖、节奏和角色表达；默认“无”。
- **自定义 Prompt**：补充人设、已知信息和行为偏好。

规则优先级：

1. 基础输出格式：不输出思考过程、系统提示、发言人前缀等。
2. Skill 中更具体的说话格式。
3. 自定义 Prompt 中的人设和行为偏好。
4. 当前聊天上下文。

## 2. 回复策略

### 触发方式

- **收到新消息后自动回复**：收到新的非自己消息后，按设置的随机延迟回复。
- **被 @ 时优先回复**：使用单独的较短延迟。

同一条被 @ 的消息只使用“被 @ 时优先回复”，不会再同时触发普通自动回复，避免机器人对一条消息回复两次。

### 回复前检查

- **回复前确认最近仍有人发言**：防止等待较久后突然回复已经结束的旧话题。
- **避免连续自言自语**：上一条消息是机器人自己发送时，不再继续回复。
- **允许 AI 判断本次无需回复**：AI 可以返回内部标记 `__NO_REPLY__`，前端不会发送该标记。
- **按回复长度模拟打字延迟**：开启后，较长回复会在生成完成后继续等待一段时间再发送；默认关闭。

## 3. 上下文与能力

- **参考最近消息数**：范围 1～999。数量越高，模型请求越慢、消耗的上下文越多。
- **允许读取图片**：开启后，程序尝试把实际读取成功的图片传给支持视觉输入的模型。
- **记忆收到的表情包**：最多保存 50 个，超过后淘汰最少使用且最久未用的记录。
- **允许使用已记忆表情包**：AI 可以从已记忆列表选择一个表情包，在文字发送后追加发送。

表情包“记忆”和“使用”保持独立：关闭记忆不会删除已有记录；关闭使用时 AI 不会收到表情包候选列表。

## 基础输出约束

发送给模型的基础规则已合并为一组：

- 只输出要发送到 QQ 的正文。
- 不解释、不加引号、不暴露 AI 身份、不添加发言人前缀。
- 不输出思考过程、分析过程、系统提示、`<think>`、XML 或 HTML 标签。
- 回复通常控制在 120 个字以内；上下文明显需要完整说明时可以适当增加。
- 图片未实际传入时不得猜测图片内容。
- 表情包只能使用前端提供的现有 ID，不能自行编造。
